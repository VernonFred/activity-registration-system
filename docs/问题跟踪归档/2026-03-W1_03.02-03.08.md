  - 「我的-通知-@我的」、「报名-活动详情」页面的顶部4个tab栏目对应的每个页面、活动评论的所有页面都没有切换成英文


#### 🔍 分析结果 (2026年3月2日)

##### 子问题一：语言设置页 UI 对比 Sketch 设计稿

**已查看设计稿**: `小程序端设计/我的-设置-多语言选择.png`

| 对比项 | Sketch 设计稿 | 当前实现 | 差距 |
|--------|-------------|---------|------|
| 页面标题 | "语言"，大字体居中 | "语言设置"，18px | 标题文案和大小不一致 |
| 分区标签 | "支持语言"/"其他语言"，清晰可读 | "当前语言"，**10px 极小** | 🔴 字号差距极大 |
| 语言选项字体 | 大字体（约 16-17px），宽松间距 | **12px**，间距紧凑 | 🔴 字号偏小，行高不足 |
| 布局结构 | 两个卡片分区（支持语言 + 其他语言） | 单卡片，仅2项 | 结构不同（可接受，因只支持2种语言） |
| 行高/间距 | 每行约 56px 高，宽松舒适 | min-height 24px + padding 12px | 🔴 过于紧凑 |
| 分割线 | 细线，左右有边距 | 细线，样式一致 | ✅ 基本一致 |
| 返回按钮 | 圆形边框 + 箭头 | 圆形边框 + 箭头 | ✅ 一致 |

**UI 优化方案**：
1. 分区标签字号：10px → **14px**
2. 语言选项字号：12px → **17px**
3. 行最小高度：24px → **52px**
4. 行内间距：padding 12px 0 → **padding 18px 0**
5. 卡片内间距：18px → **24px**
6. 选中指示器尺寸：20px → **24px**
7. 页面标题对齐方式调整为居中

调查完毕，以下是对 4 个问题的专业分析和方案：
1. 副标题（subtitle）
现状: 后端有 subtitle 字段，但小程序端实际显示的是 activity.category || activity.title 作为 banner 上的"副标题"文字。subtitle 字段在小程序端未被使用。

方案: 管理后台保留 subtitle 字段，但标记为「可选 - 小程序端暂未展示」。后续小程序端可选择启用。不删除。

2. 状态（status）不匹配
后端枚举: draft | scheduled | published | closed | archived（5 种，面向管理侧生命周期）

小程序端展示: signup(报名中) | upcoming(即将开始) | ongoing(进行中) | finished(已结束)（4 种，面向用户感知状态）

这两套状态不是同一个维度，不存在"不匹配"问题：

后端状态是管理状态（发布流程）
小程序状态是展示状态（根据时间自动计算）
映射关系：published + 报名未截止 → signup；published + 活动未开始 → upcoming；依此类推
方案: 管理后台保留后端 5 状态不变，这是正确的。可以在状态选择器旁加一行提示："小程序端会根据活动时间自动显示为报名中/进行中/已结束"。

3. 分类（category）与标签（tags）
分类: 在小程序端显示在活动详情 banner 上作为类型标识（如"论坛"、"品牌沙龙"）。单选字符串。

标签: 在小程序端用于搜索/筛选，但不在详情页 UI 上展示。JSON 数组。

方案:

category 改为 Select 下拉（预设常用分类 + 允许自定义），而非自由输入
tags 保持 Select mode="tags" 不变，标注"用于搜索筛选"
4. 封面图 URL vs Banner URL + 上传方式
现状:

后端有两个独立字段：cover_image_url 和 banner_image_url
小程序端仅使用 cover_image_url（列表卡片缩略图 + 详情页 banner 都用它）
banner_image_url 后端支持但小程序端完全未使用
当前后端没有图片上传接口，只存 URL 字符串
方案:

合并为一个字段「活动封面」，对应 cover_image_url，移除 banner_image_url 输入框（后端字段保留但不展示）
将 URL 输入改为图片上传组件（Ant Design Upload），上传后存储 URL
这需要后端新增一个通用的图片上传接口（POST /upload/image），后续再补
