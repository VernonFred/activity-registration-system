# 会议报名小程序产品规划

## 1. 项目概述
- **项目背景**：品牌部门举办的论坛、沙龙类活动频率不高，但需要高效完成邀请、报名、审核、签到、反馈等流程，避免线下 Excel 和纸质名单的重复劳动。
- **目标用户**：高校老师（参会者）和品牌部门管理员（运营者，单一角色）。
- **产品愿景**：提供一体化的轻量报名管理工具，覆盖活动宣传、报名审核、现场签到和会后反馈，确保数据沉淀与复用。
- **平台选择**：以微信小程序为主入口，兼顾扫码/分享便捷性；若后期需要，可基于同一前端框架输出 H5 辅助入口。

## 2. 使用场景
1. 老师通过邀请链接/二维码查看活动、报名、查看审核结果和签到码。
2. 管理员创建活动、配置报名表单、审核报名、推送通知、导出数据及复盘。
3. 会后管理员发放电子徽章和满意度问卷，沉淀参会记录。

## 3. 功能架构
```
微信小程序端
├── 首页（活动探索）
├── 报名（活动列表 & 详情）
├── 消息（系统通知）
└── 我的
    ├── 我的资料
    ├── 我的报名
    │   ├── 待审核
    │   ├── 已通过/待签到
    │   ├── 已参加
    ├── 徽章墙
    └── 设置（反馈、客服）

管理端（Web）
├── 概览仪表盘
├── 活动管理
├── 报名表单设置
├── 报名审核
├── 通知中心
├── 签到管理
├── 数据导出与报表
└── 徽章管理/系统设置
```

## 4. 微信小程序端规划
### 4.1 导航结构
- 底部 Tab：`首页`、`报名`、`我的`。
- 消息中心入口置于顶部消息图标或 `我的` 页内醒目位置。

### 4.2 首页
- **顶部搜索 & 标签快速筛选**：按会议系列（如“高博会”“湘江论坛”）切换。
- **Banner**：轮播重点活动或回放。
- **活动推荐列表**：卡片展示活动封面、标题、时间、地点、报名状态（报名中/名额已满/报名未开始）。
- **内容聚合**：可扩展往期回顾文章或精彩瞬间，增强品牌曝光。

### 4.3 报名 Tab
- **近期活动**：默认按时间正序排列，可过滤标签、城市、是否需要审核。
- **历史活动**：用于回顾资料、查看回放链接、下载会议资料。
- **活动详情页**：
  - 活动摘要（时间地点、报名截止、联系人）。
  - 议程/嘉宾、往届精彩（富文本）。
  - 报名进度条与剩余名额。
  - `立即报名` 按钮，若已报名则展示状态与操作（查看审核结果、修改报名信息、签到二维码、上传缴费凭证）。
  - 收藏/分享功能。

### 4.4 报名流程
1. **填写基础资料**：首次填写姓名、单位、职称、手机号、邮箱、备注；支持选择是否同行人员、是否住宿等自定义字段。
2. **附加问题**：可根据活动配置选择题或问答题。
3. **提交前确认**：信息确认弹窗，勾选《活动条款》。
4. **提交 & 状态提示**：显示“待审核”，并推送通知；若活动无需审核则直接 `报名成功` 并生成签到码。
5. **修改/撤回**：在报名截止前允许修改资料或撤回报名。

#### 4.4.1 默认报名表单字段建议
- **个人资料（必填）**：姓名、学校、部门/职务、手机、邮箱。
- **行程安排**：
  - 必填：接站点（下拉选择）、到会时间（日期时间选择器）、送站点、返程时间。
  - 选填：到会车次/航班号、返程车次/航班号。
- **住宿安排（必填）**：是否需要会议安排住宿（单选：自行/会务安排）、拟入住酒店（下拉选择）、住宿意向（单人/合住）、房型偏好（大床/标准间）。
- **其他选项（根据活动需求配置）**：是否随行嘉宾、饮食禁忌、发票需求、特殊备注等。
- 所有字段仍需在后台支持必填/选填开关，以应对特殊活动灵活调整。

#### 4.4.2 默认下拉选项（长沙示例）
- **接站点/送站点**：长沙黄花国际机场 T2、长沙南站高铁、长沙火车站、长沙汽车西站、长沙汽车南站。
- **到会时间/返程时间**：日期选择范围默认覆盖活动前后各 3 天，时间段按 30 分钟粒度。
- **拟入住酒店**：长沙瑞吉酒店、长沙北辰洲际酒店、长沙尼依格罗酒店、长沙国金万丽酒店、长沙马栏山华天智选酒店。
- **住宿意向/房型**：保持与字段一致的单选项（单人/合住、大床/标准间）。
- 默认选项可在后台活动配置中增删，以满足其他城市或专场需求。

### 4.5 我的 Tab
- **个人信息卡片**：头像、姓名、单位；点击可编辑基本资料。
- **我的报名**：按状态分组，支持快捷操作：
  - `待审核`：展示预计审核时间，可补充材料。
  - `已通过`：显示签到二维码、活动提醒、补充材料入口。
  - `已参加`：查看现场签到时间、评分与反馈按钮。
- **徽章墙**：展示已获得的会议徽章（按会议系列或参与次数解锁），提升用户荣誉感。
- **设置**：意见反馈、联系客服、关于我们、隐私协议。

> **官方活动群入群指引**：在“我的报名”中，当状态为“已通过”时展示专属区域放置微信官方活动群二维码，用户可直接长按识别加入群聊；二维码需支持按活动单独配置，跨活动可复用历史二维码。

### 4.6 消息中心
- 列表展现系统通知：报名受理、审核结果、补资料提醒、签到提醒、会后感谢及问卷。
- 点击跳转至对应页面（如审核结果→报名详情，问卷→评价页）。

### 4.7 会后反馈与徽章
- 活动结束后推送评分页：星级评分+文字点评。
- 徽章自动发放规则示例：首次参会、连续三次参会、参加特定系列活动。
- 徽章展示在 `我的` 页面，可分享生成海报（可选迭代）。

### 4.8 设计风格指南
- **小程序端**：整体沿用苹果系简洁、留白充足的设计，结合毛玻璃/半透明卡片背景；主色建议采用品牌蓝 + 中性灰，配合柔和投影；图标保持线性风格。
- **管理端 Web**：参考提供的仪表盘示例，打造浅色系 + 明亮强调色（橙或品牌蓝）的卡片式布局；侧边栏固定导航、内容区域使用圆角卡片与柔和阴影，确保视觉统一且信息层级清晰。
- **组件复用**：前端开发阶段把常用模块（卡片、按钮、Tab、统计卡）沉淀为可配置组件，保证后续迭代效率。

## 5. 管理端规划（单管理员）
### 5.1 概览仪表盘
- 今日/近期活动数量、报名人数、已审核通过率、签到率概览。
- 待处理任务提醒（待审核报名、待推送通知）。

### 5.2 活动管理
- **创建/编辑活动**：基本信息、活动时间段、地点、人数限制、报名起止时间、是否需要审核/缴费、是否开放later评分。
- **内容编辑**：富文本编辑议程、嘉宾介绍、资料附件链接。
- **活动状态控制**：草稿、预发布、发布中、结束、归档。
- **复制活动**：快速创建同类型活动。
- **官方群二维码**：每个活动可上传/刷新微信群二维码图片，支持设置失效提醒，方便同步至前端展示。

### 5.3 报名表单设置
- 默认字段模板（姓名、单位、职务、手机号、邮箱等）。
- 可配置额外字段（多选/单选/填空）、设置必填项、提示文案。
- 支持保存为模板，未来新活动复用。
- 针对出行与住宿信息提供预设字段包（接送站、航班车次、住宿安排、房型等），管理员可一键启用或自定义。

### 5.4 报名审核
- 报名列表：显示个人信息、报名时间、自定义字段回答。
- 批量通过/驳回，支持备注原因。
- 审核操作触发通知（微信订阅消息+短信/邮件备用）。
- 查看操作历史，避免误操作。

### 5.5 通知中心
- 发送站内消息/微信订阅消息模板：报名成功、审核结果、签到提醒、活动变更、感谢信。
- 支持定时发送（例如活动前一天提醒）。
- 模板管理：维护消息标题、正文变量（姓名、活动时间等）。

> **微信群二维码管理**：在活动详情中上传或更新官方微信群二维码（含生效期提醒），并支持生成带活动信息的引导图片，确保用户在小程序端能看到最新的入群二维码。

### 5.6 签到管理
- 生成活动签到二维码（可下载打印）。
- 实时查看签到人数、缺席名单。
- 支持现场手动补签或扫码核销。

### 5.7 数据导出与报表
- 导出报名名单（Excel/CSV），可选择字段。
- 活动统计：报名人数、审核通过率、签到率、评分平均值。
- 用户统计：参会次数、活跃度、徽章数量。

### 5.8 徽章管理
- 创建徽章（名称、图标、描述）。
- 配置发放规则（参与次数、特定活动、评分≥4 等）。
- 手动发放/撤回。

### 5.9 系统设置
- 管理员账号信息、角色权限（单账号即可）。
- 隐私协议与条款编辑。
- 日志记录（活动发布、审核操作、数据导出行为）。

## 6. 关键业务流程
1. **活动发布流程**：创建活动 → 配置表单 → 发布 → 推送邀请。
2. **报名流程**：用户填写资料 → 提交 → （可选）管理员审核 → 通知结果 → 若通过则生成签到码。
3. **通知流程**：系统事件触发（审核通过、活动变更、提醒） → 通知中心发送 → 用户消息中心查看。
4. **签到流程**：现场扫码/管理员核销 → 记录签到时间 → 更新状态。
5. **会后复盘**：推送评分 → 收集反馈 → 导出数据 → 生成徽章 → 活动归档。

## 7. 数据模型核心字段（初稿）
- `User`：openId、姓名、单位、职称、手机号、邮箱、徽章列表。
- `Activity`：标题、标签、封面、时间、地点、人数限制、报名窗口、状态。
- `ActivityContent`：议程、嘉宾、资料链接（富文本）。
- `FormField`：类型、是否必填、选项、排序。
- `Signup`：用户、活动、填写字段数据（JSON）、状态（待审核/通过/驳回/已取消）、审核备注、签到状态。
- `Notification`：类型、发送渠道、模板变量、发送状态。
- `CheckinRecord`：签到时间、方式、操作人。
- `Badge`：名称、图标、规则、发放时间。
- `Rating`：评分值、文字反馈、是否公开。

## 8. 技术实现建议
- **前端**：
  - 微信小程序端选用 `Taro + React` 或 `uni-app + Vue3`，可一套代码输出微信小程序与 H5（作为备用）。
  - UI 组件库：TDesign/ColorUI/uView 等，保证表单与列表样式统一。
- **管理端**：
  - 使用同一框架生成 Web 前端（Taro H5 / Vue + Element Plus）。
  - 登录通过账号密码 + 简单验证码；未来可接入企业微信扫码登录。
- **后端**：
  - Python 技栈推荐 `FastAPI`（轻量、易拓展）或 `Django`（自带后台管理）。
  - 数据库：MySQL 或 PostgreSQL；使用 SQLAlchemy/Pydantic 做数据模型。
  - 文件存储：对象存储（如腾讯 COS）用于活动封面、徽章图标、报名附件。
  - 消息服务：微信订阅消息（需申请模板），可选短信/邮件作为备用渠道。
- **部署**：
  - Docker 容器化，Nginx 反向代理 HTTPS。
  - 定时任务（Celery/APScheduler）处理定时通知、徽章发放。
  - 日志与监控：应用日志、异常告警、数据库备份。

## 9. 里程碑规划
- **MVP（约 4 周）**：
  1. 管理端：活动管理、报名表单配置、审核、导出。
  2. 小程序端：活动列表/详情、报名表单、报名状态、消息列表。
  3. 基础通知：微信订阅消息（报名成功、审核结果）、短信备用。
- **迭代 2（约 3 周）**：
  - 签到二维码、实时统计、later评分、徽章体系初版。
  - 管理端仪表盘、通知定时发送。
- **迭代 3（约 3 周）**：
  - 历史活动回顾、资料中心、问卷扩展（多题型）。
  - 数据报表、用户画像、模板分享。

## 10. 运维与安全
- 所有敏感信息（手机号、邮箱）加密存储，登录与接口使用 HTTPS。
- 权限控制：仅管理员账号可访问后台，配合操作日志和导出记录。
- 数据备份策略：每日自动备份数据库至安全存储，保留 30 天。
- 隐私合规：提供隐私协议、数据使用说明；活动报名需告知用途。
